Everything You Need

Powerful features in a single binary

🎨

Web UI

Dark theme interface to visually manage all your mock routes. No config files needed.

📜

JavaScript Scripts

Write dynamic responses with full JavaScript. Access method, path, params, query, and body.

📥

Swagger Import

Import OpenAPI 2.0/3.x specs and auto-generate mock routes instantly.

🔌

WebSocket Mock

Mock WebSocket endpoints with auto-reply or JavaScript message handlers.

⚡

GraphQL Support

Mock GraphQL queries and mutations with operation-based matching.

🔄

Proxy Mode

Forward unmatched requests to real backend. Perfect for incremental mocking.

⚙️

Conditional Routes

Match by headers or body content. Different responses for different requests.

🔀

Dynamic Paths

Path parameters with :id syntax. Wildcards with * for catch-all routes.

🚀

Zero Dependencies

Single Go binary. No Node.js, no database, no runtime dependencies.

Simple Yet Powerful

See how easy it is to create mock APIs

# Create a route via API
curl -X POST http://localhost:8088/_api/routes \
  -H "Content-Type: application/json" \
  -d '{
    "method": "GET",
    "path": "/users/:id",
    "status": 200,
    "body": "{\"id\": {{id}}, \"name\": \"Alice\"}"
  }'

# Test it
curl http://localhost:8088/mock/users/42
# {"id": 42, "name": "Alice"}
# Dynamic response with JavaScript
curl -X POST http://localhost:8088/_api/routes \
  -H "Content-Type: application/json" \
  -d '{
    "method": "GET",
    "path": "/random",
    "script": "respond({
      status: 200,
      body: JSON.stringify({
        rand: Math.floor(Math.random() * 1000),
        timestamp: Date.now()
      })
    })"
  }'

# Each request returns different random number
# Create GraphQL mock
curl -X POST http://localhost:8088/_api/graphql \
  -H "Content-Type: application/json" \
  -d '{
    "operationName": "GetUsers",
    "response": {
      "users": [
        {"id": 1, "name": "Alice"},
        {"id": 2, "name": "Bob"}
      ]
    }
  }'

# Query it
curl -X POST http://localhost:8088/graphql \
  -H "Content-Type: application/json" \
  -d '{"query": "{ GetUsers { id name } }"}'
# Import from OpenAPI spec
curl -X POST http://localhost:8088/_api/import-swagger \
  -H "Content-Type: application/json" \
  -d @openapi.json

# All paths auto-generated as mock routes!
# Supports OpenAPI 2.0 (Swagger) and 3.x
# JSON and YAML formats
